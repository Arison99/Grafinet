Background
Software systems continuously evolve to accommodate new requirements, fix defects, improve performance, and adapt to changing technologies. GitHub has become one of the most important platforms for managing software evolution through version control, collaboration, issue tracking, and continuous integration.
This assignment is designed to help you develop practical skills in using GitHub effectively while understanding its role in supporting software evolution.
Assignment Tasks
Part A: Repository Creation and Setup (10 Marks)
1.	Create a public GitHub repository named:
Software-Evolution-Assignment
2.	Add the following files:
o	README.md
o	LICENSE
o	CONTRIBUTING.md
3.	Write a professional README containing:
o	Project title
o	Project description
o	Installation instructions
o	Usage instructions
o	Contributors section
Part B: Version Control and Commit Management (20 Marks)
Create a simple software project of your choice (e.g., calculator, student management system, task manager, inventory system).
Perform at least 10 meaningful commits over several days.
Requirements:
•	Each commit message must be descriptive.
•	Avoid generic messages such as "updated file" or "fixed bug".
•	Demonstrate incremental development.
Examples:
•	Added user authentication module
•	Implemented search functionality
•	Fixed validation error in registration form
Part C: Branching and Merging (15 Marks)
1.	Create at least two feature branches:
o	feature-ui
o	feature-enhancement
2.	Make changes in each branch.
3.	Merge both branches into the main branch using Pull Requests.
4.	Resolve at least one merge conflict and document the resolution process in the README.
Part D: Issue Tracking and Project Management (15 Marks)
Create at least:
•	5 GitHub Issues
•	1 GitHub Milestone
The issues should represent realistic software evolution activities such as:
•	Bug fixes
•	New features
•	Refactoring tasks
•	Documentation improvements
Close at least three issues through commits or pull requests.
Part E: Software Evolution Analysis Report (30 Marks)
Prepare a report (1500–2000 words) addressing the following:
1. Introduction
•	Brief description of the project.
2. GitHub Features Used
Discuss:
•	Repositories
•	Commits
•	Branches
•	Pull Requests
•	Issues
•	Milestones
3. Software Evolution Support
Explain how GitHub facilitates:
•	Corrective maintenance
•	Adaptive maintenance
•	Perfective maintenance
•	Preventive maintenance
4. Challenges Encountered
Describe challenges experienced while using GitHub and how they were resolved.
5. Reflection
Discuss lessons learned and how GitHub supports collaborative software evolution.
