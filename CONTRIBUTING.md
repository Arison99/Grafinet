# Contributing

Thanks for your interest in contributing to Grafinet.

## Workflow

1. Fork the repository and create a feature branch
2. Use clear commit messages for each incremental change
3. Open a Pull Request into main with a clear description
4. Reference related issues in PR and commit messages

## Code Standards

- Keep components small and focused
- Validate all backend inputs
- Handle frontend error states clearly
- Update docs when behavior changes

## Pull Request Checklist

- Feature is tested locally
- No sensitive data is committed
- README is updated if needed
- Issue references are included when applicable
